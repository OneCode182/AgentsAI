# Memory

> Durable project knowledge that persists across sessions: conventions, decisions, patterns, mistakes, and module status.

## Contents

| File | Purpose | When to load |
|---|---|---|
| `conventions.md` | Naming, file structure, language, code style, and commit conventions | Any implementation task — always relevant |
| `decisions.md` | Architecture Decision Records (ADRs) and significant technical choices | Before proposing architectural changes; after making one |
| `patterns.md` | Proven implementation patterns reused across the codebase | Feature implementation, refactoring |
| `mistakes.md` | Known pitfalls, past bugs, and anti-patterns to avoid | Bug fixes, code review, QA validation |
| `module-status.md` | Current state of each product module: done, in-progress, planned, blocked | Task planning, scope assessment, orchestration |
| `prompts/` | Archived original human prompts and routing metadata | Prompt intake, similarity search, session resume decisions |

## Access Pattern

1. `prompts/_.index.md` is loaded by `workflows/prompt-intake.workflow.md` before substantial task decomposition.
2. `conventions.md` is near-universal — load for any code-touching task.
3. `decisions.md` and `patterns.md` are loaded for new features and refactors.
4. `mistakes.md` is loaded for bug fixes and review tasks.
5. `module-status.md` is loaded by the orchestrator for planning and prioritization.

## Contributing

- Memory files are **append-friendly** — add new entries at the end of the relevant section.
- Each entry should be atomic: one convention, one decision, one pattern, one mistake.
- Use consistent entry format within each file (follow existing entries as template).
- Date-stamp decisions and mistakes: `YYYY-MM-DD`.
- Never delete entries — mark obsolete ones with `[DEPRECATED]` and reason.
- Prompt archives use `prompts/YYYY-MM-DD-{short-name}.prompt.md` and preserve the original prompt.

## Rules

- Memory is the lowest-authority source — it informs but does not override protocols, specs, or architecture.
- Agents should consult memory before implementation to avoid repeating known mistakes.
- After completing a session that produces new knowledge, the agent must append to the relevant memory file.
- Memory files must stay in Spanish for business-domain terms and English for technical terms (per `conventions.md` language rules).
- Prompt archives may preserve the human's original language verbatim.
